# 🛡️ 양방향 퀀트 가드 (Quant Guard) 자동매매 시스템 최종 운영 명세서

본 문서는 **AWS Lambda**, **Amazon S3**, **한국투자증권 OpenAPI (KIS)**, 그리고 **Telegram Bot API**를 결합하여 구축된 양방향 모멘텀 퀀트 자동매매 시스템의 최종 완료 사양 및 운영 명세서입니다. 

기존 소스코드의 파편화된 가이드를 통합하고, 실전 가동 중 발견된 라이브러리 의존성 에러(requests 패치) 및 보안 수정을 모두 거친 최신 릴리즈 버전을 기준으로 작성되었습니다.

## 🏗️ 2. 시스템 아키텍처 및 파이프라인

본 시스템은 안정성과 실행 속도를 극대화하기 위해 데이터 수집/연산 계층과 주문 집행 계층이 물리적으로 철저히 분리된 **서버리스 파이프라인(Serverless Pipeline)** 구조를 따릅니다.

```text
[매주 월 15:05] Lambda A (시그널 엔진) ──> KRX 데이터 수집 & 퀀트 스코어링
                                           │
                                           ▼ (JSON 암호문 업로드)
                                   [Amazon S3 금고]
                                           │
                                           ▼ (JSON 신선도 검증 후 다운)
[매주 월 15:15] Lambda B (주문 엔진) ───> 한투 API 잔고 조회 & 리밸런싱 주문 집행
                                           │
                                           ▼ (실전 / 테스트 분기 영수증 발송)
                                    [텔레그램 스마트폰 알림]
```

## 🔑 3. AWS Lambda 환경 변수 마스터 가이드

모든 핵심 제어 권한 및 증권사/텔레그램 비밀키는 소스코드를 절대 직접 수정하지 않고, AWS Lambda 콘솔의 **[구성] ➔ [환경 변수]** 창에서 원격 관리하여 보안 유출을 원천 방어합니다.

### 🔴 최상위 컨트롤 가드 (안전핀)
* **`FORCE_TEST_MODE`**: `True` 또는 `False` 값 주입
  * **`True` (가상 리허설):** 모든 계좌 연산과 시뮬레이션은 실전과 100% 똑같이 돌지만, 최종 한투 서버로의 주문 전송만 차단(Mocking)한 뒤 스마트폰으로 `🧪 테스트 모드 영수증`을 발송합니다.
  * **`False` (실전 가동):** 가상 보호막을 끄고 한국투자증권 실전 매매 통로를 개설하여 진짜 돈으로 주식을 사고팝니다.
* **`CASH_RESERVE`**: 원 단위 숫자 입력 (예: `1000000` 입력 시 100만 원 격리)
  * 은성님의 예수금 중 봇이 절대 매수하지 못하도록 잠가두는 현금 방화벽입니다. 총자산에서 이 금액을 자동으로 제외한 자산으로만 포트폴리오를 구성합니다. (`0` 입력 시 예수금 전액 투자)

### 🔵 증권사 및 인프라 연동 변수
* **`APP_KEY`**: 한국투자증권 KIS Developers에서 발급받은 실전투자용 APP Key
* **`APP_SECRET`**: 한국투자증권 KIS Developers에서 발급받은 실전투자용 APP Secret
* **`ACCOUNT_NO`**: 한국투자증권 종합주식 계좌번호 앞 8자리 숫자
* **`TELEGRAM_TOKEN`**: 텔레그램 `@BotFather`를 통해 생성한 봇의 HTTP API 고유 토큰 문자열
* **`TELEGRAM_CHAT_ID`**: 은성님의 개인 대화방 고유 숫자 ID (봇방 진입 후 활성화 필수)

## 🕒 4. 월요일 실전 운영 타임라인 (Timeline)

매주 월요일 장 마감 직전, 전산 시스템이 스스로 깨어나 10분 간격으로 토크 리밸런싱을 마감합니다.

### 1단계: 15:05 (KST) ➔ Lambda A 가동 [시그널 생성 및 대뇌 역할]
* **휴장일 교차 검증:** KIS API 및 휴장 정보망을 호출하여 당일이 휴장일인 경우 즉시 프로세스를 안전 안전 종료합니다.
* **유니버스 필터링:** 국내 상장된 전체 ETF 목록(약 1,100여 개)을 조회한 후 레버리지, 인버스, 곱버스, 채권, MMF, 단기금융, 달러/엔화 환노출, 환헤지((H), 선물H) 키워드를 가진 파생 및 안전자산 ETF를 사전에 원천 배제합니다.
* **유동성 컷오프:** 시장 왜곡 및 슬리피지 방지를 위해 필터링된 유니버스 중 거래대금 상위 100개 종목만 최종 압축합니다.
* **모멘텀 랭킹 산출:** 126 영업일(약 6개월) 가격 데이터 가속도를 연산하여 최종 스코어를 부여합니다.
* **섹터 중복 방어 변산 투자:** 20개로 정의된 섹터 키워드 사전을 바탕으로 분류를 수행하며, 섹터당 가장 모멘텀이 높은 1개 종목만 채택하여 최대 10개 종목을 최종 선정합니다.
* **마켓 가드 (VIX 필터):** 야후 파이낸스에서 미국 변동성 지수(^VIX)를 조회하여 임계치(30.0) 이하면 **BULL(상승장 리밸런싱)**, 초과면 **BEAR(하락장 전량대피)** 상태코드를 부여하여 S3 버킷에 `quant_signals.json` 파일로 구워 올립니다.
* **비서 리포트:** 스마트폰 텔레그램으로 "📊 오늘의 퀀트 모멘텀 랭킹 보고서"를 즉시 전송합니다.

### 2단계: 15:15 (KST) ➔ Lambda B 가동 [주문 집행 및 손발 역할]
* **시그널 신선도 가드:** S3 금고에서 데이터를 내려받을 때 파일 생성 타임스탬프를 체크하여 1시간 이상 지난 과거 데이터일 경우 주문 오작동 방지를 위해 즉시 프로세스를 셧다운합니다.
* **실시간 계좌 평가조회:** 한투 API 서버를 통해 실시간 총자산 평가액 및 주문 가용 예수금을 원 단위로 정확하게 파싱합니다.
* **가용 예산 확정:** 실시간 총자산에서 환경 변수로 지정된 `CASH_RESERVE` 액수를 차감하여 실제 주식 매매에 투입할 순수 예산을 계산합니다.
* **억까 차단 방어선 (자산 0원 가드):** 계좌에 돈이 0원이거나 투자 원금을 모두 빼둔 경우(`investable_asset <= 0`), 코드 에러로 팅기지 않고 `ZERO_ASSET_COMPLETED` 분기 통로를 안전하게 타고 주문 수량을 0건으로 확정, 즉시 다음 공정으로 패스합니다.
* **비중 차액(Diff) 연산 및 주문 집행:** 목표 10개 종목의 목표 비중과 내 계좌의 현재 보유 비중을 1원 단위까지 대조하여 `매도` 주문을 먼저 전송한 후, 확보된 현금으로 `매수` 주문을 KIS 최우선 지정가 호가 규격(5원 단위 정밀 가드 반영)에 맞춰 전송합니다.
* **테스트 모드 우회:** `FORCE_TEST_MODE = True`일 시, 이 주문 전송 단계만 모킹(Mock) 시뮬레이션으로 안전하게 우회 처리합니다.
* **최종 텔레그램 실시간 영수증:** 프로세스가 안전 종료되면 은성님 스마트폰으로 투자 비율, 매매 내역, 체결 보고를 담은 정밀 자동매매 완료 영수증을 즉시 쏘아 올립니다.

## 🚨 5. 위급 상황 대처 및 비상 제어 매뉴얼 (Operations)

본 시스템은 돌발적인 시장 폭락이나 개인적인 자금 사정 변경 시 코딩 지식 없이도 3초 만에 시스템을 통제할 수 있도록 비상 프로토콜이 탑재되어 있습니다.

### 1. 시장 급변 시 즉시 무인 공장 전원 차단 스위치
* **상황:** 장중에 갑작스러운 대형 악재가 터져 월요일 오후 리밸런싱 주문을 강제로 막고 싶을 때
* **조치:** AWS Lambda B 콘솔 ➔ 환경 변수 창에서 `FORCE_TEST_MODE` 값을 즉시 **`True`** 로 수정 후 저장합니다.
* **효과:** EventBridge 알람시계에 의해 오후 3시 15분에 봇이 강제로 깨어나더라도, 실제 주문 집행 통로가 완벽하게 마비되어 실제 원금은 1원도 나가지 않고 가상 시뮬레이션 보고서만 발송됩니다.

### 2. 실전 운용 자금 실시간 변경 프로토콜
* **상황:** 계좌에 들어있는 돈 중 일부를 개인적인 사정으로 출금해야 하거나, 봇이 사지 못하게 급하게 비상금으로 묶어둬야 할 때
* **조치:** 환경 변수 `CASH_RESERVE` 칸에 격리할 현금 금액을 숫자로만 적고 저장합니다. (예: 250만 원을 묶으려면 `2500000`)
* **효과:** 봇이 잔고를 파싱할 때 해당 금액은 없는 돈 셈 치고 격리벽 뒤로 숨겨두며, 남은 자산으로만 안전하게 소액 분산 리밸런싱을 진행합니다.

### 3. 베타테스터식 실전 출격 주말 체크리스트
정책(슈퍼 ISA 등) 도입 시차와 OpenAPI 지원 여부를 주말 간 조율하면서 최종 실전 가동을 위해 월요일 아침 장 시작 전에 세팅해야 할 마스터 조건입니다.
1. **Lambda B `config.py` 확인:** `FORCE_TEST_MODE = False` 상태로 배포(Deploy) 마감 확인
2. **실전 투자 원금 이체:** 월요일 오전 출근길에 한국투자증권 일반 계좌(자동매매 신청 완료 계좌)로 베타테스트 소액 투자 원금 입금 완료하기
3. **일정 조율 팁:** 만약 주말이나 월요일 아침 뉴스에서 오픈 API를 전면 허용하는 신규 조세 특례 계좌 소식이 나온다면, 입금을 즉시 보류하고 람다 B 환경 변수의 `ACCOUNT_NO` 지갑 주소만 새 계좌로 바꾼 후 입금하시면 세금 혜택까지 100% 챙기며 무인 공장을 가동할 수 있습니다.
